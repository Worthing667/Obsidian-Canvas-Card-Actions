# 技术实现：卡片合并

本文档说明 `Canvas Card Actions` 当前版本中“卡片合并 / 预览工作台”功能的实现方式。这里记录的是已经落地的代码结构，而不是早期方案草稿。

## 1. 当前交互结构

多选卡片后的主交互已经收敛为三条入口：

- `一键复制`
- `一键拼合`
- `打开预览...`

对应关系如下：

- `一键复制`：按设置 `defaultSortMode` 直接生成文本并写入剪贴板
- `一键拼合`：按设置 `defaultSortMode` 直接在当前 Canvas 新建卡片
- `打开预览...`：打开标准 `ItemView` 工作台，在其中切换排序、预览结果，并执行输出

旧的合并命令仍然保留在命令面板中：

- `MergeToCanvasCardCommand`
- `MergeToSidebarPreviewCommand`
- `MergeToMarkdownCommand`
- `ManualMergeCommand`

但它们已经不再挂载到多选右键菜单。

## 3. 命令层设计

### 3.1 快速命令

`src/presentation/commands/QuickActionCommands.ts`

新增了三类快速命令：

- `QuickCopyCommand`
- `QuickMergeCommand`
- `OpenPreviewWorkbenchCommand`

它们的特点是：

- 都接收当前选区快照来源
- 都读取统一设置 `defaultSortMode`
- 都不再让右键菜单同时暴露“排序方式”和“输出方式”两个维度

### 3.2 旧命令兼容

`src/presentation/commands/MergeCommands.ts`

旧命令仍保留，原因是：

- 命令面板和快捷键用户仍然可以直接调用
- `ManualMergeCommand` 仍复用 `DragSortModal`
- 兼容过渡阶段不会强行打断旧使用习惯

其中 `MergeToSidebarPreviewCommand` 的内部实现已经改为打开新的工作台视图，而不是旧的极简文本预览。

## 4. 统一的内容生成层

`src/services/ContentService.ts`

这是这次改造的核心。新的实现不再只返回“拼好的字符串”，而是同时支持“先拿到有序卡片列表，再决定如何展示或输出”。

### 4.1 关键接口

- `createSelectionSnapshot(selection)`
- `getOrderedCards(options)`
- `buildMergedContent(options)`
- `copyMergedContent(options, successNotice)`

### 4.2 选区快照

`createSelectionSnapshot` 会把当前选区转换为纯数据快照，包含：

- `id`
- `text`
- `x`
- `y`
- `width`
- `height`
- `badge`
- `badgeType`

这一步的意义是：

- 工作台内部不依赖 live selection
- 工作台被拖到独立窗口后仍能工作
- 排序切换、手动调整、预览渲染都基于同一份稳定输入

### 4.3 排序能力

`getOrderedCards` 统一支持三种模式：

- `position`
- `badge`
- `manual`

具体策略仍然复用领域层：

- `PositionSortStrategy`
- `BadgeSortStrategy`

`manual` 模式则根据 `manualOrderIds` 对快照重新排序。

### 4.4 输出内容构建

`buildMergedContent` 只负责根据排序结果返回：

- `content`
- `count`

当排序模式是 `badge` 且允许徽章前缀时，输出格式为：

```text
[徽章] 内容
```

其余模式输出纯文本段落，并用空行分隔。

## 5. 工作台状态服务

`src/services/PreviewWorkbenchService.ts`

这个服务负责维护工作台的一次性状态，而不是让 `ItemView` 自己承载全部业务逻辑。

### 5.1 状态结构

`src/types/WorkbenchState.ts`

当前工作台状态包含：

- `canvasFilePath`
- `canvasFileBasename`
- `selectionSnapshot`
- `sortMode`
- `manualOrderIds`
- `previewExpanded`
- `lastComputedContent`

### 5.2 主要职责

- 创建初始状态 `createState`
- 切换排序模式 `setSortMode`
- 维护手动拖拽顺序 `reorderManual`
- 控制预览区展开状态 `setPreviewExpanded`
- 生成当前顺序对应的预览文本 `buildPreviewContent`

### 5.3 手动模式继承

这次改造里最关键的一点是：

- 从 `位置` 切到 `手动` 时，继承当前位置排序结果
- 从 `徽章` 切到 `手动` 时，继承当前徽章排序结果

因此“手动排序”不再是一个独立功能，而是对当前自动排序结果的微调。

## 6. 工作台视图实现

`src/presentation/views/MergeWorkbenchView.ts`

工作台基于 Obsidian 标准 `ItemView` 实现，类型仍沿用：

```ts
MERGE_PREVIEW_VIEW_TYPE = 'canvas-card-actions-merge-preview'
```

这让它天然支持：

- 在侧边栏打开
- 被拖成独立窗口
- 作为普通 tab 使用

### 6.1 三段式布局

当前视图分为三块：

1. 顶部工具栏
2. 中部顺序列表
3. 底部预览与输出区

### 6.2 顶部工具栏

顶部展示：

- 排序模式切换按钮：`位置 / 徽章 / 手动`
- 当前 Canvas 名称
- 选区快照数量
- 当前模式和可输出卡片数

### 6.3 中部顺序列表

列表展示当前排序结果：

- `位置 / 徽章` 模式下只读
- `手动` 模式下支持拖拽

实现方式是视图内部监听原生拖拽事件：

- `dragstart`
- `dragover`
- `drop`
- `dragend`

拖拽完成后，通过 `PreviewWorkbenchService.reorderManual` 更新 `manualOrderIds`。

### 6.4 底部预览区

预览区默认折叠，展开后才开始生成文本。

当前已经落地的性能策略：

- 默认折叠
- 展开后再生成全文
- 预览更新做了 `200ms` 防抖
- 卡片数量超过阈值时显示“内容较多，展开后再渲染”的提示

### 6.5 输出按钮

工作台底部统一提供三个动作：

- `复制`
- `新建卡片`
- `新建文稿`

这三者都基于“当前工作台顺序”生成内容，只是输出目标不同。

## 7. MergeService 的职责变化

`src/services/MergeService.ts`

在旧实现中，`MergeService` 更偏向“拿当前 selection 直接输出”。现在它扩展成了两个层次：

- 直接处理当前选区
- 处理来自工作台的选区快照

### 7.1 对外接口

当前主要接口包括：

- `mergeToCanvasCard`
- `mergeToSidebar`
- `mergeToMarkdown`
- `openWorkbench`
- `mergeSnapshotsToCanvasCard`
- `mergeSnapshotsToMarkdown`

### 7.2 打开工作台

`openWorkbench` 会做这些事：

1. 读取当前选区快照
2. 根据 `defaultSortMode` 创建初始工作台状态
3. 激活或复用右侧 `ItemView`
4. 把复制 / 新建卡片 / 新建文稿的回调注入到视图上下文

### 7.3 自动恢复源 Canvas

工作台里的 `新建卡片` 不再依赖“原 Canvas 必须正处于激活状态”。

当前实现流程是：

1. 记录 `canvasFilePath`
2. 优先查找当前 workspace 已打开的对应 Canvas leaf
3. 如果没找到，则尝试自动打开对应 `.canvas`
4. 拿到新的 Canvas 实例后再执行 `addNode`

只有在这些情况下才会失败并提示：

- `.canvas` 文件不存在
- 文件路径失效
- 自动打开后仍未拿到 Canvas 实例

## 8. 设置项迁移

这次改造把旧设置：

- `mergeDefaultOrder`

迁移为：

- `defaultSortMode`

涉及文件：

- `src/settings/ICanvasCardActionsSettings.ts`
- `src/settings/CanvasCardActionsSettingTab.ts`
- `src/adapters/StorageAdapter.ts`

### 8.1 迁移策略

`StorageAdapter.loadSettings()` 中做了兼容迁移：

- 旧数据里如果存在 `mergeDefaultOrder`
- 且新字段 `defaultSortMode` 不存在
- 则自动把旧值映射到新字段

这样旧用户升级后不需要重新配置。

## 11. 回归建议

建议手测以下路径：

1. 多选后执行 `一键复制`
2. 多选后执行 `一键拼合`
3. 打开工作台后切换 `位置 / 徽章 / 手动`
4. 从 `位置` 切到 `手动` 后拖拽，确认顺序继承正确
5. 工作台执行 `复制 / 新建卡片 / 新建文稿`
6. 关闭原 Canvas 后，从工作台执行 `新建卡片`
7. 选中大量卡片时确认预览默认折叠且展开后内容正确
