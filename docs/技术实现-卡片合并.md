# 技术实现：卡片合并

本文档说明 `Canvas Card Actions` 中“卡片合并”功能的实现细节。

当前能力覆盖三种输出：

- 合并并新建 Canvas 卡片
- 合并并在侧边栏预览
- 合并并新建 Markdown 文稿

同时支持“一键合并（默认）”，其行为由设置项决定。

## 1. 菜单入口与命令挂载

卡片合并挂载在多选菜单事件 `canvas:selection-menu` 上，与复制能力同一入口。

关键位置：`src/main.ts`

- `registerCanvasMenus`：监听多选菜单事件并读取 `canvas.selection`
- `addSelectionMenuCommands`：注册并挂载合并命令

新增菜单项：

- `一键合并（默认）`
- `合并并新建卡片`
- `合并并侧边栏预览`
- `合并并新建文稿`

对应命令类位于：`src/presentation/commands/MergeCommands.ts`

- `MergeByDefaultCommand`
- `MergeToCanvasCardCommand`
- `MergeToSidebarPreviewCommand`
- `MergeToMarkdownCommand`

## 2. 生成与输出解耦

核心实现是把“内容生成”和“输出目标”解耦：

1. `ContentService` 负责生成合并文本
2. `MergeService` 负责将同一份结果分发到不同输出

### 2.1 ContentService：统一合并构建

关键位置：`src/services/ContentService.ts`

新增能力：

- `buildMergedContent(options)`：统一返回 `{ content, count }`
- `resolveSelection(selection)`：优先使用传入 selection，否则回退到当前 Canvas 选区

同时复用原有排序策略：

- 位置排序：`PositionSortStrategy`
- 徽章排序：`BadgeSortStrategy`

这也修复了旧逻辑中“传入 selection 但内部未使用”的一致性问题。

### 2.2 MergeService：输出编排

关键位置：`src/services/MergeService.ts`

职责：

- `mergeByDefault`：按设置路由到默认输出
- `mergeToCanvasCard`：落到白板卡片
- `mergeToSidebar`：写入侧边栏视图
- `mergeToMarkdown`：写入 Vault 文稿

## 3. 三种输出实现

### 3.1 新建 Canvas 合并卡片

流程：

1. 调用 `buildMergedContent` 获得文本
2. 计算锚点卡片（按位置找到第一个文本卡）
3. 在锚点右侧偏移创建新文本卡
4. `canvasAdapter.addNode` + `requestSave`

关键点：

- 新卡默认继承锚点宽高
- 位置偏移固定为 `anchor.width + 40`

### 3.2 侧边栏预览

新增视图：`src/presentation/views/MergePreviewView.ts`

- `MERGE_PREVIEW_VIEW_TYPE = 'canvas-card-actions-merge-preview'`
- `setContent({ content, count, order })` 用于渲染内容和元信息

在 `src/main.ts` 中：

- `registerMergePreviewView` 注册 `ItemView`
- 通过 `MergeService.activateMergePreviewView` 打开或复用右侧 leaf

### 3.3 新建 Markdown 文稿

新增适配器：`src/adapters/VaultAdapter.ts`

- `createMergedDocument(content, canvasFile, baseName)`
- 默认保存在当前 `.canvas` 同目录
- 文件名格式：`<canvas名>-卡片合并-时间戳.md`
- 若冲突，自动追加 `-1/-2/...`

## 4. 设置项扩展

设置接口：`src/settings/ICanvasCardActionsSettings.ts`

新增：

- `mergeDefaultOrder: 'position' | 'badge'`
- `mergeDefaultOutput: 'canvas-card' | 'sidebar-preview' | 'markdown-file'`

设置页：`src/settings/CanvasCardActionsSettingTab.ts`

新增两项下拉：

- `卡片合并默认顺序`
- `卡片合并默认输出`

默认值定义：`src/main.ts` 的 `DEFAULT_SETTINGS`

- `mergeDefaultOrder: 'position'`
- `mergeDefaultOutput: 'canvas-card'`

## 6. 关键交互与回归点

建议手测路径：

1. 选中多张文本卡，执行 `一键合并（默认）`
2. 执行 `合并并新建卡片`
3. 执行 `合并并侧边栏预览`
4. 执行 `合并并新建文稿`
5. 切换设置中的默认顺序与默认输出后复测 `一键合并（默认）`
6. 回归旧能力：
   - 按位置复制
   - 按徽章顺序复制
   - 手动排序复制
   - 拆分卡片
   - 徽章编辑
