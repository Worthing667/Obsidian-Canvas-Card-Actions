根据您提供的代码分析，问题在于插件的菜单注册逻辑中存在不一致性。当选择多张卡片时，属性查看功能通过 `canvas:selection-menu` 事件注册，但对于单张卡片，插件使用 `canvas:node-menu` 事件，而在该事件处理器中没有添加属性查看功能。

## 问题原因

在 `addNodeMenuCommands` 方法中，插件只为单个节点添加了以下菜单项：
- 徽章管理功能
- 拆分卡片功能  
- 复制单卡内容功能

但缺少了"查看卡片属性"的选项。

## 解决方案

需要在 `addNodeMenuCommands` 方法中添加属性查看命令。以下是具体的修改方案：## 完整的修改指导

您需要在 `main.ts` 文件中的 `addNodeMenuCommands` 方法中添加属性查看功能。具体修改如下：

```
// 在 addNodeMenuCommands 方法中添加以下代码
// 位置：在现有命令添加之后，方法结束之前

addNodeMenuCommands(menu, node) {
  // ... 现有的徽章、拆分、复制命令 ...

  // 添加属性查看功能 - 适用于所有文本节点
  if (node.getData && node.getData().type === "text") {
    // 添加分隔符以区分不同类型的命令
    menu.addSeparator();
    
    const propertiesCommand = new OpenCardPropertiesCommand(
      this.app,
      this.cardService,
      [node] // 将单个节点包装为数组
    );
    
    this.commandRegistry.registerCommand("open-single-card-properties", propertiesCommand);
    this.commandRegistry.addCommandToMenu(
      menu, 
      "open-single-card-properties", 
      "查看卡片属性...", 
      "info"
    );

    // 可选：同时添加复制尺寸功能
    const copyDimensionsCommand = new CopyCardDimensionsCommand([node]);
    this.commandRegistry.registerCommand("copy-single-card-dimensions", copyDimensionsCommand);
    this.commandRegistry.addCommandToMenu(
      menu,
      "copy-single-card-dimensions",
      "复制卡片尺寸",
      "copy"
    );
  }
}
```

## 关键要点

**统一性保障**：这个修改确保单卡和多卡选择都能使用相同的属性查看功能，提供一致的用户体验。

**命令标识符区分**：使用不同的命令ID（`open-single-card-properties` vs `open-card-properties`）避免冲突，但实际使用相同的Command类。

**节点类型验证**：添加了适当的条件检查，确保只有文本类型的节点才显示属性查看选项。
