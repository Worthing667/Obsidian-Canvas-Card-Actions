# 功能：卡片内容复制与排序

本插件当前把多卡片内容处理分成两条主线：

- 快速通道：`一键复制`、`一键拼合`
- 完整工作台：`打开预览...`

这样用户在大多数场景下只需要先决定“要不要一步完成”，而不是先在右键菜单里同时判断排序方式和输出方式。

## 1. 单张卡片复制

右键点击单张文本卡片时，可以使用：

- `复制卡片内容`

行为：

- 直接复制当前卡片的纯文本内容
- 不涉及排序

## 2. 多张卡片的一键复制

多选文本卡片后，可以使用：

- `一键复制`

行为：

- 读取当前选区
- 按设置 `一键排序方式` 排序
- 直接复制到剪贴板

当前 `一键排序方式` 支持：

- `按位置顺序`
- `按徽章顺序`

如果设置为按徽章顺序，输出格式为：

```text
[徽章] 内容
```

## 3. 多张卡片的一键拼合

多选文本卡片后，可以使用：

- `一键拼合`

行为：

- 读取当前选区
- 按设置 `一键排序方式` 排序
- 直接在当前 Canvas 新建一张合并卡片

新卡片会创建在当前选区锚点卡片右侧，并继承其宽高。

## 4. 预览工作台

多选文本卡片后，可以使用：

- `打开预览...`

它会打开一个标准的 Obsidian 侧边栏视图。这个视图也可以被用户原生拖成独立窗口。

### 4.1 工作台能做什么

工作台统一支持：

- 切换排序模式
- 查看当前顺序
- 展开查看最终合并文本
- 复制结果
- 新建 Canvas 卡片
- 新建 Markdown 文稿

### 4.2 排序模式

工作台里有三种排序模式：

- `位置`
- `徽章`
- `手动`

规则如下：

- `位置`：自动按空间位置排序，只读
- `徽章`：自动按徽章排序，只读
- `手动`：允许拖拽调整顺序

当你从 `位置` 或 `徽章` 切换到 `手动` 时，手动列表会继承当前排序结果作为起点，而不是重新从头开始。

### 4.3 预览区

底部预览区默认折叠。

展开后会显示当前顺序对应的最终文本结果。这样可以在输出前先检查合并效果。

当卡片很多或内容很长时，默认折叠可以减少不必要的渲染开销。

## 5. 当前推荐用法

### 快速场景

适合直接使用右键菜单：

- 只是想把当前白板顺序复制出去
- 只是想快速生成一张合并卡片

### 需要确认或微调顺序的场景

适合先打开工作台：

- 想在位置排序基础上略微调整顺序
- 想对徽章排序结果做人工修正
- 想先检查最终合并文本
- 想输出为 Markdown 文稿

## 6. 设置项

当前与多卡片复制/拼合直接相关的设置项有两个：

- `设置卡片排序优先级`
- `一键排序方式`

### 6.1 设置卡片排序优先级

只影响“按位置排序”的细节：

- 优先按垂直方向排序
- 优先按水平方向排序

### 6.2 一键排序方式

影响：

- `一键复制`
- `一键拼合`
- `打开预览...` 时的默认排序模式

可选项：

- `按位置顺序`
- `按徽章顺序`

## 7. 命令面板兼容

虽然右键菜单已经简化，但旧能力并没有完全移除。

命令面板里仍保留：

- 合并为新卡片
- 在工作台中预览
- 合并为新文稿
- 手动排序拼合

这主要是为了方便已经绑定快捷键的用户继续使用。

## 8. 典型使用场景

- 会议纪要整理：按白板位置快速复制
- 任务状态汇总：按徽章顺序输出
- 写作结构编排：进入工作台后手动拖拽微调
- 归档草稿：在工作台里确认文本后新建 Markdown 文稿
